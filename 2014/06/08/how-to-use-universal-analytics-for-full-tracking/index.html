如何用Universal Analytics对网站实施完整监测

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

由于Universal Analytics(以下简称UA)在网站分析领域已是大势所趋，必将完全取代目前的Google Analytics，所以接下来我要整理的主要是基于UA的配置与各种常见问题解决方案，希望帮助初学者快速入门并与有经验人士探讨更多可能。
因为本网站是基于Wordpress平台搭建的,所以下面所说的例子会结合我自己的网站以及Wordpress的结构特点进行部署。大家请记住，几乎所有的CMS系统原理都是差不多的，所以不管你是Wordpress,还是Disqus，Jekyll,Hexo等等，一种通了则其他的也用类似的方法实现即可，不必对此钻牛角尖，毕竟最重要的是实现的思路方法。

基础配置

  1. Universal Analytics账号配置
  2. Universal Analytics代码生成与部署
  3. 测试代码，验证数据

完善细节

  1. 明确转化，设置目标
  2. 去除多余URL参数
  3. 识别新增搜索引擎关键词
  4. 监测站内搜索(In site Search)
  5. 监控网站服务器是否正常
  6. 数据警报(网站数据异常提醒)
  7. 定时发送数据
