多图博客福音-七牛图床

发表于 2017-02-07 分类于 分享 Waline： 本文字数： 621 阅读时长 ≈ 2 分钟

像我们这种把网址挂在github上本来就已经是hard浏览模式：访问速度比国内网址慢了好多.虽然依托hexo的纯静态处理，已经把访问速度提升到了极致，但是服务器在国外，很多时候访问仍然不是那么顺畅。但是在我经历了国内备案，香港服务器间歇性崩盘之后，发现github pages确实是最有良心的，也不好再去奢求更多了，毕竟稳定>快速。

又一次安利完，说一下这次主要的网站图片问题。我的上一篇《2016总结》，里面的图有50来张，尽管我都压缩过，原照许多在3-5M，都被压缩到1M以内。但尽管如此，50多张图片加起来也有20-30M，对于本来就很慢的github服务器来说，无疑又是一记毁灭性的重拳。因此把图片存在本地确实是非常不明智之举，这时候图床的出现真的是茫茫大海中的一盏明灯。

我用的是极简图床，页面和操作交互都是我喜欢的极简风格。之间的差别我不细讲，举个例子，打开我上面提到的文章页，时间1/30吧，比例我随便说的，但是实际上比例确实有这么夸张。我一个炫图的文章页打开3分钟了还是一片空白，确实是说不过去。感谢七牛为我们提供了这么好的服务。

极简图床免费版一个小时内只能上传50张图片，很有限，但是通过七牛账号个人认证后就会取消该限制并获得10G的存储空间，所以尽量还是不要使用阉割版，很难受，使用教程传送门。

对于Hexo用户来讲，更有趣的是，所有图片上传完，不仅能给到html，js等格式的链接，同时还提供markdown格式，一键复制，甚为体贴，虽然只是一个小细节，但是可以感受到这位PM的用心，赞一个。

为什么说这么多，一张介绍的图都没有。因为我懒…很多问题答案都留在链接里了，别人已经写过的我只要保存链接然后外链一下就好，利己利人（我平生找了很多借口，这也是我比较满意的一个）。
