生活需要仪式感

发表于 2017-02-07 | 分类于 纪录 |

小王子驯养狐狸后，第二天又去看它。

“你每天最好相同时间来。”狐狸说，

小王子问：“为什么？”

“比如，你下午四点来，那么从三点起，我就开始感到幸福。时间越临近，我就越感到幸福。我就发现了幸福的价值……所以应当有一定仪式。”

“仪式是什么？”小王子问。

狐狸说，“（仪式）它就是使某一天与其他日子不同，使某一时刻与其他时刻不同。”

《小王子》中的片段，最近很流行仪式感这个词，两年前其实看完这本书并没有多大感触。这次回家过年却明显感觉到为什么仪式感那么重要。大家之所以觉得过年越来越没有年味了，不是因为外界环境、风俗变化等等，而是因为自己本身对过年这件事情就缺少了重视、缺少了仪式感。把过年就当成家人聚在一起吃饭，吃完饭看春晚，打牌，如是而已。

Python与数据

发表于 2016-12-30 | 分类于 纪录 |
  • Numpy
    • 强大的ndarray对象和ufunc函数
    • 精巧的函数
    • 比较适合线性代数和随机数处理等科学计算
    • 有效的通用多维数据,可定义任意数据类型
    • 无缝对接数据库
  • Scipy library
    • Python中科学计算程序的核心包
    • 有效计算numpy矩阵,让NumPy和SciPy协同工作
    • 致力于科学计算中常见问题的各个工具箱,其不同子模 块有不同的应用,如插值、积分、优化和图像处理等
  • Matplotlib
    • 基于NumPy
    • 二维绘图库,简单快速地生成曲线图、直方图和散点图等形式的图
    • 常用的pyplot是一个简单提供类似MATLAB接口的模块
  • Pandas
    • 基于 SciPy 和 NumPy
    • 高效的Series和DataFrame数据结构
    • 强大的可扩展数据操作与分析的Python库
    • 高效处理大数据集的切片等功能
    • 提供优化库功能读写多种文件格式,如CSV、HDF5

福利-微信小程序UI标准

发表于 2016-11-07 | 分类于 分享 ， 产品 |

我们一直都说，“产品为王”，“内容为王”，但是这些话就如同“我找对象外貌不要紧，最重要的是有才华”一样扯淡。这是一个看脸的时代，在任何地方都适用。产品的外观和交互设计也是产品竞争力的一个重要体现，就和上面找对象的例子一样，你看着都丑谁还看你有没有内涵？

最近准备研究一下小程序，惊喜地发现微信和小程序提供的UI标准文档，俨然一块金矿！在这里你可以深刻感觉到，大公司对于交互和视觉方面的用心，在做产品设计的这条路上，他就是一套标准的教科书，比起市面上其他的收费课程，简直是业界良心。

废话不多说，先甩两个链接

  • 微信小程序设计指南
  • WeUI样式库

具体暂不讨论，大家去链接出看原版即可，必有收获，可以根据其目录管中窥豹：

  • 友好礼貌
    • 重点突出
    • 流程明确
  • 清晰明确
    • 导航明确，来去自如
    • 减少等待，反馈及时
    • 异常可控，右路可退
  • 便捷优雅
    • 减少输入
    • 避免误操作
    • 利用接口提升性能
  • 统一稳定
  • 视觉规范
    • 字体规范
    • 列表视觉规范
    • 表单输入视觉规范
    • 按钮使用原则
    • 图标使用原则

每一个产品，在设计的时候如果都能把以上元素全部考虑进去，在不讨论功能差异的情况下，我认为就可以是一个比较完整的产品了。因为这不仅仅是外观，同时也是对产品的态度。
