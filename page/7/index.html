Universal Analytics监测细节的完善(六)——数据警报

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

上一篇介绍了后端状况的监控设置，这一次我们需要通过设置对数据的异常情况得到比较及时的反馈。

一般广告公司或互联网公司运营部门，都是整天和数据打交道的。而其中一项很重要的职责是对于访问数据中的异常得到非常及时的反馈并找到针对性的措施解决相应的问题。而在广告公司更不幸运的是，有时候你一个人会管理几个甚至十几个网站的GA账户，而未必能每天都对着GA账户一个一个地去看，那么怎么才能比较高效地在这么多账户中及时地发现问题呢？这就需要用到GA的Alert预警功能。

打开UA后台，找到Admin-View-Custom Alert

如图按提示填写相应的内容，选定相应的维度和指标，当指标符合某类特定条件时(绝对值大于或小于特定值,变化的比率大于或小于特定值)，则向你发送邮件预警，提醒你数据有比较明显变化，是该看看后台了。(希望我能收到某天100 session的警告)

为了让用户更加方便地使用这个功能，Google非常人性化地加上了重复关联(图中的Apply)的功能，即如果你需要监控一系列站点的同一类数据，那么你只需设置一次，然后关联到其他的账户中，如果出现问题发送预警，GA会在邮件中以列表的形式呈现出来，非常方便。

Universal Analytics监测细节的完善(五)——网站服务器是否正常 - 监控宝

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

除了数据之外，你的网站服务器是否稳定也是一个非常重要的因素。作为个人站长来说，不可能每时每刻甚至说每天去查看自己的网站，那么我们就需要一些工具或者平台的辅助，定时地帮我们测试网站访问是否正常(而且我们还不想花钱),那么**监控宝**是你非常好的选择。

监控宝是由北京云智慧科技所提供的一项非免费服务，可以定时监控网站、服务和服务器器的可用率（Uptime）和响应时间(Response Time)，一旦网站无法连结、Apache等Web服务器负载过高、Mysql等数据库压力过大、或是服务器发生错误，监控宝服务就会在最短的时间内以Email或简讯来通知网站管理员，让因网站停摆而造成的损失降到最低。监控宝同时支持基于线路和地区的监控告警设置，可以用来较好的应对中国复杂的网络状况和环境，帮助网站管理员更全面的了解和管理网站。

除此之外，该服务也提供实用的「使用者访问速度追踪」功能，可以追踪国内不同地区的使用者进入你网站的速度。
对免费用户，监控宝提供5个监测需求，你只需输入你需要监测的请求或者链接，设置好相应的时间间隔，那么监控宝服务器则会按照设置的时间间隔无限地进行请求并返回结果，若某一次请求失败或者超时，则会立刻发送短信或者E-mail通知你，最大限度地减少服务器问题带来的影响和损失。

我使用的是免备案的香港服务器，相对于内地服务器来说部是那么地稳定。所以更加需要这类的服务，因为这家供应商及客服态度非常好，只要你收到监控宝的邮件说打不开，就可以去找他们聊天了。

另外如果你或者你的企业有需要的话，可以购买监控宝的收费服务，反应更快，功能更强，其中一大亮点是可以通过授权同步GA中的数据，可以将访问、负载情况与流量数据进行结合，这个是非常有趣的事情，感兴趣的朋友可以试一试。对于企业来说，最贵的套餐也不过￥599/月，确实不贵。

用了他家免费的功能好几年，一直没有给与经济上的支持，所以在这里相当于给他打一个广告把，虽然我的博客影响力及其有限，但是轻声细语也可以是感谢，对吧。

Universal Analytics监测细节的完善(四)——监测站内搜索

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

站内搜索一直是一块非常有价值的资源，也是容易被大多数人遗忘的宝藏。你的网站内容结构好不好，是否满足用户的需求，如何发掘用户其他潜在的需求，在用户的搜索内容中会得到很多的体现。所以，监测好用户站内搜索的内容是网站分析中不可或缺的一环。

那么在UA中如何设置站内搜索的监测呢？其实也比较简单，其实在GA中就早已有此功能，因为不起眼所以很少获得关注。下面我们来设置一下：

首先，确定搜索参数

如同添加默认识别搜索引擎一样，需要监测站内搜索的内容，首先你需要知道你搜索的内容存在什么位置、什么参数中。举我的网站作为例子，当我搜索google analytics的时候，搜索结果页的URL为：

http://jackie.ms/?s=google+analytics&submit=%E6%90%9C%E7%B4%A2

其中s就是我需要的参数 (补充一点，Wordpress站点的搜索参数都未s)

其次，后台补充设置

打开Admin-View-View Setting

打开Site Search Tracking，并将该参数填入相应位置，保存即可。
(最后选项为在content报告中URL是否去除搜索词参数)

补充，数据查看

在Reporting报告界面中，找到Behavior-Site Search即可查看相应站内搜索词，从这个时刻开始，你会更加了解你的用户，他们喜欢看什么内容，还想看什么内容，然后试着去补充这方面的内容把，相信用户会更加喜欢你的网站的。

Universal Analytics监测细节的完善(二) —— 去除多余参数

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

假如你在内容报告上看到以下类似情况：

同一个页面由于后面各种乱七八糟的参数，导致被分割成几个或者多至数千个页面，这样你在面板中看到的数据会非常混乱，后期在手动处理数据尤其是做多维度较差的时候会平添很多麻烦。为了防止这种事情发生，我们需要前期做好沟通与测试，因为这些参数可能来自内部参数传递也可能是外部广告参数。

内部参数传递 : 在站内页面与页面之间的切换中可能为了内部的分析会传递一些id，产品型号，或者内部搜索参数等等，你只需要从技术同事那里得到一份完整的列表，再添加进下图中的位置即可

外部广告参数: 如果你用过秒针检测系统就会知道，他家的广告监测链接后面会追加一大票的参数例如mz_sp,mz_XX等等，有时因为他们参数太长超过了广告投放平台的限制导致了部分问题的发生。同样这些参数也可能会影响到UA报表的呈现。

解决这个问题的办法很简单，确实UA本身就提供了这种功能，找到Admin - View - View Settings 中的Exclude URL Query Parameters

按图中格式填好，保存即可，记得参数之间用半角英文隔开，格式错误的话设置无效。（之前的中文说明坑了不少人，中文翻译参数中间是用句号隔开的，所以鉴于翻译效果不好把控，尽量使用英文版把）

Universal Analytics监测细节的完善(一) —— 明确转化，设置目标

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

借毛主席的话“一切不以结婚为目的的谈恋爱都是耍流氓”，我认为“没有目标的网站分析都是瞎扯淡”。一开始做网站分析的时候，很容易走入一个误区，哪些数据有意思，哪些来源很诡异，找出原因后会有一些成就感，如同解出高中变态物理题一般，而随后在乐此不疲的干这些事情。事情的过程比结果有意思，但是结果才是价值的体现。所以明确你网站的目的是非常重要而急切的事情，所以我把它摆在了第一位。

1. 定位自己的网站

你的网站是个什么类型的网站，能提供什么内容或者服务，用户的什么行为能给你带来收益？

2. 关键转化点

每个网站归根结底就是一个或两个关键的转化点，比如电商就是销售产品，门户网站就是流量广告变现，博客就是扩大影响力，活动网站就是参与或者注册……搞清楚自己最想要什么，那么你的思路就会更加清晰。

3. 清晰转化路径

在转化前一般会经历哪些必要环节，可能有哪些路径？把他们一一列举出来，找到所有的步骤。

4. 在UA中设置Goal及相关参数

来到Admin - View - Goal 界面

有四种Goal可以供你选择：

Goal 说明
Destination 目标页面
Duration 停留时间
Page/Screens per Session 每次访问查看页面数
Event 事件(视频播放，按钮点击)
这里需要注意的是，如果你要看到报表中的Goal Funnel(目标转化漏斗),那么请一定把这个Goal选择为Destination,或者在该行为发生时触发虚拟页面监测代码，其他类型都不支持Funnel

5. 数据分析

利用得到的数据去验证你的假设，猜想可以提供方向，但是数据更能告诉你猜想的结论。同时通过各个环节的流失数据，我们可以找到哪些环节出了问题，哪些路径更加直接有效，哪些来源的用户目标达成率较高，达成目标的用户有着哪些共通的属性或者行为？

6. 行动、优化

像之前说的，结果才是最重要的，你经历了这么多设置，收集这么多数据，得到了那么多结论，通过这些结论采取了哪些措施，并最终是否带来了提升？有多少的提升？如果上面两个问题回答不了，那将会是一个不愉快的体验。

Universal Analytics的账户结构

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics

这里了解一下Universal Analytics(以下简称UA)的结构,从管理界面上来看，它可以分为三个等级：Account账户 - Property网站资源 - View(在GA为profile配置文件)

这三个级别其实就是由上到下的层级关系，举个简单的例子：我这有2个网站：一个是自己的博客Jackie.ms和Fashionton(朋友经营的时尚网站),那么我的账户就是Jackie,我有2个Property,分别就是这两个网站，而在Jackie.ms中，我想有多个视图View, 分别可以看到所有数据、来自搜索引擎的数据、来自微博的数据、来自朋友友链的数据，这么一来结构就清楚了：

  • Jackie Account

    • Jackie.ms Property

        *   所有数据 `View`

      • 来自搜索引擎的数据 View

      • 微博数据 View

      • 友链 View

      • Fashionton.com Property

        • 所有数据 View

      • 微博 View

      • 贴吧 View
        根据你不同的需求，可以选择不一样的配置，一切为自己的方便着想，不用遵循特定的规矩。但是一点需要注意：不管你有多少个视图，做了多少配置，请永远保留一个未作任何配置的原始视图，因为一旦你过度自信，配置过滤出了问题，那么中间丢失的数据可能就永远找不回来了。

昨天看了一部非常好的电影《恐怖直播》，在看之前有些犹豫，因为毕竟是韩国的片子，也许是受韩剧的影响，我一直以为他们也仅仅是喜欢骗骗少女的尖叫或者眼泪而已。之前看过的《我的野蛮女友》、《非常主播》这一类的片子还是挺有意思的，但是在心底我可能还是觉得他们的电影很low，拍不出震撼人心的大片的。但是看完这部，我对韩国电影的印象已经彻底改变。

片中每个人物的性格都特别鲜明，而男主角的演技也是堪称一绝，和我之前印象中韩剧主演没事长个嘴夸张地说不出话来有着极大的差别。里面所有人物都在为自己的利益斗争，“恐怖分子”为冤死父亲讨回公道采取极端手法谋求关注；主播为恢复原职隐瞒恐怖袭击线索，只为抢独家；主管为升职提升收视率不惜以人质性命作代价；国家安全部长官立场坚定不予妥协最后被爆头惨死……最后的结局也出人意料，因为男主角从最开始的“意外收获”，到后来的“参与其中”，再到后来的被同事曝光受贿黑史变得“无法逃脱”，再到被告知成为整个事件的替罪羊，到最后替恐怖分子按下了炸弹按钮以获得最后的解脱，短短几个小时内，让他从一个落寞的主播变成了炸毁整座大楼的“恐怖分子”。把利益至上与人情冷漠刻画地非常的真实，也把ZF在这件事情处理上产生的问题显露无疑。

虽然这一个半小时的剧情都是在一个演播室里发生的，但是效果完全不逊于其他商业大片。他说的不是一个特别的故事，而更像是说的是发生在身边的事实。在我们周围也总会听过或看过各种不公，各种利益争斗，有许多人“报官无门”，不能从合法途径维护自身的利益，因而采取一些极端的措施。而ZF为了维持自己的威信，就算牺牲一些人的性命作为代价也不能对“恐怖分子”妥协，不管他的要求是否合理，因为从你引爆第一颗炸弹开始，你自身的性质就不一样了，从“维权者”变成了“恐怖分子”。影片从头至尾让男主角在这条道路上重走了一遍，所以最后的结局看起来也是必然，因为经过这一切公众视线已经完全被转移，从“维权，要求总理道歉”到“恐怖分子炸弹袭击”、“主播收受贿赂，为获奖抢妻子新闻”。而我们可以想象第二天全国铺天盖地的新闻是一些什么内容，不出意外的话官媒应该是强烈谴责这种暴徒行为，ZF安慰伤者，当局当机立断态度强硬，成功击毙犯罪嫌疑人。娱乐媒体可能更多的去挖掘男主播黑史，而造成这一切的原因，不管是法制问题还是腐败问题，都显得不是那么的重要……而这些是更应该让我们去思考的。

这部电影被“合情合理”地列为了禁片，相信没人会感到意外，甚至连审片的朋友估计都会觉得惋惜，但是还是强烈推荐大家一看。顺便查了一下这个影片的获奖情况：

Wordpress中用Markdown语法写博客

发表于 2014-05-14 更新于 2016-09-21 分类于 wordpress

什么？你不知道Markdown语法，那你真是Out了！它是目前机器写作最高大上的方式，允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML(或者HTML)文档，因其简洁的语法加上强大的展现方式流行于程序员的圈子，而且正在猛烈地向外扩散，想知道它具体能做什么，请戳【Wiki百科介绍】，【语法介绍】，只能说程序员不可怕，文艺的程序员才可怕，比如史蒂夫.乔布斯。

关于语法的介绍我这里不多写，Google一下遍地都是，在这里我要说的是怎么在wordpress中使用wordpress语法便捷地写出更“漂亮”的文章(当然，这里的漂亮指的是外表)。很多人第一时间应该都会想到，既然有这个需求那么wordpress应该会有相应的插件了把？是的，已经有很多支持Markdown语法写作的wordpress插件了，一搜也有一大把。

上面的几个插件我都一一试过了，别看几个评价都是5星，但是我的结果是：没有一个好用。可能是我的要求比较严苛，他们暂时还达不到我的要求，如果你持有怀疑态度的话，可以一试，如果觉得还凑合的话可以用用。据我所知还有一部分高手正在开发更好的插件，我们暂且等一段时间看看。

那么既然不用插件我们用什么工具方便一点呢？这里我推荐一个“曲线救国”的方式，使用下面两个工具：

  • Stackedit.io

  • 马克飞象

他们的界面如下：

 

 

这两个都是在线Markdown编辑器，其中两者各有优劣：

Stackedit:

  • 支持多格式导出(PDF,MD,HTML)
  • 一键分享或存储到Dropbox或者Google Drive

马克飞象

  • 对中文有更好的体验
  • 多主题选择(我数了一下，目前支持24种主题，而Stackedit为3种，都不好看)
  • 一键同步到Evernote(对了，Evernote就是只大象) 但是请注意你使用的是Evernote国际版还是印象笔记，两者是不一样的
  • 即时查看和复制HTML代码
    他们的主要区别就是上面标红的部分，所以如果你只是正常写博客，推荐使用马克飞象，在它的界面中写好以后直接转成HTML代码，然后在发布文章的时候选择【文本】格式即可。而且你可以同时一键同步到你的Evernote账户下，作为一个快速备份，一举两得。

但是如果你想要直接产出文件发给他人，那么Stackedit就相当方便了，根据需求选择。

也有人提到过用本地工具SublimeText或者Mac下的Mou，我很不推荐，但是也欢迎你尝试，我对这两者的看法是：

SublimeText

非常好用的代码编辑器，但是Markdown需要安装插件，且不能实时显示效果，每做一个修改都要按几个键才能预览到效果，

Mou

语法比较落后，经常显示错误的效果，开始用的话你会怀疑是否你的语法写错了。但是这个时候请相信自己，这么简单的语法还能写错？
它对于我的功能其实就是一个本地文件保存器，因为时不时需要在本地保存.md文件，比如我的Github页面,每一篇文章都是在后台deploy一个md文件到github上。

发表于 2014-05-12 更新于 2016-09-21 分类于 纪录

标题这几个字很能反映我自己的状态，对好多事情都有兴趣，都想好好玩玩，认真地玩。但是最后结果是大多数都停留在一个比较浅的阶段。比如说编程、音乐、魔方、运动、玩游戏，或者是折腾折腾网站，写写博客，研究研究GA,Doublelick,GTM，感觉我脑子还是比较好使，能比较快地进入角色或在短时间内从初学者蜕变成一个中级水平。但是如果往高级了去，需要花费更多的时间和精力而只获得之前同时间相比相差较大的”收益”时，我就会从潜意识里”认怂”，进而把时间留到其他的兴趣爱好中去。在别人看起来好像感觉好多东西都会一些，好像很厉害的样子，但是在我自己感觉上，总觉得自己好像走入了一个误区。因为总是不确定自己真正喜欢做的是什么，今后自己一定还会做的是什么，如果以后不会做了，那么现在花太多时间进去显得有点浪费，从而一直在一个循环中走不出来。

我自认为还是一个比较勤快的人吧，很怕浪费自己的时间，因此最不喜欢的事情之一就是等待,并不是说我觉得我的时间比较值钱，而是任何人的时间都比较宝贵，尽量少让或者不让别人等你是一种很大的尊重。即使比较空闲，我还是喜欢给自己找些事情做，不管在别人看起来是不是无聊。

很羡慕那些在很年轻的时候就能找到自己非常钟爱的事物的人，因为比起其他人来，他们少了很多犹豫与纠结，一条路走下去，能在这个领域走得更深，看到更多一般人看不到的风景。而大多数人同我一样，只是在不停地试，不停地换，不停地寻找。而随着年龄的增长，尝试新鲜事物的恐惧也会与日俱增。然后选择妥协，回归现实，做好自己目前的工作，结婚生子，防止各种”意外“的发生。

在尝试的路上，我们总是太急于看到结果，一旦一段时间过去了，虽然你有些许进步，但是你看不到这些金不带给你的价值，因为这不是游戏，打一个怪涨几点经验，点滴的变化也能清晰地展现。这个时候会很容易怀疑自己这个方向是否正确，力是否使对了地方，或者自己是不是真的喜欢，要不要坚持。而这个问题，可能时间才能给你答案。

我经常在想，自己既然知道了这种情况，应该做一点怎样的调整来改善一下这种状况？其实根本原因在于缺乏合理的计划和安排,突然想到了知乎上收藏过的一个帖子，你有什么相见恨晚的知识想推荐给年轻人？，当时看着只觉得很有道理，看完就关掉了这个页面。现在回头再看一遍，我觉得应该有所行动了

Wordpress 主题翻译—Poedit

当你遇到一款喜欢的英文主题时，如果主题里的选项中的英文每次都让你看着别扭或不习惯，那么为什么不自己翻译一下这个主题呢？
其实操作方法很简单，一个ftp工具(我再Mac上用的是transmit,Windows一般用flashXP)用来上传文件，一个翻译器(Poedit)加上一个词典(有道、google翻译等)就轻松搞定了。

第一步，拿到原始language包中的文件

连接ftp，找到如下地址/web/wp-content/themes/你的主题名/languages/，找到其中的原始语言包，一般以po或者pot结尾的文件，下载到本地。

第二步，编辑原始语言文件，开始翻译

具体操作步骤非常简单，你指需要将相应的英文在翻译区写上对应的中文即可，完成以后保存，但记得一定要勾上**偏好设置-行为-保存时自动编译mo文件**,如下图

第三步，上传覆盖

如果你**languages文件夹下仅有po文件是不能达到最终效果的，真正起作用的mo文件,另外命名的话最好和wp-config.php**文件中的<span style="color: #ff6600;">**define('WPLANG', 'zh_CN')**</span>;一致(中文界面的话一般都为zh-CN)，好习惯总是没有坏处的。

上传完成，来欣赏下自己的杰作把

如果仅仅是自己用的话，翻译的调皮点增加乐趣我也是极为推崇的。但是如果这个包要分享给其他人用的话，还是官方一点把，最后，如果你也用的是我这个Spacious主题,这里我把我翻译的语言包分享给你,祝使用愉快！
