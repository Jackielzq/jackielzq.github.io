关于金钱

发表于 2014-11-13 更新于 2023-04-23 分类于 纪录 本文字数： 1.2k 阅读时长 ≈ 4 分钟

为什么会突然想到这个问题，最近一直有同事问我房子租哪儿，我回答说，公司附近，走路10分钟到。他们首先反应的不是说这个距离这么近，而是“啊，那一定很贵吧，多少钱”。说到这里，我还真算了一笔账：

比如说我现在房租是2000/每月，如果住在稍微偏一点的地方，房租可能1300-1500，但是算一下其他成本：

1.每天交通8块，一个月计22天，176元

2.每天上下班时间各1小时，一共2小时

3.中午吃完午饭可以午睡40分钟

4.小区有免费的健身房，每天下班早可以去锻炼

5.每天可以睡到更晚

等等……

其实就1，2两点来说，其隐形价值加上房租就已经超过2000，剩下的就都是福利了。现在对我来说更宝贵的是时间，不管是拿来睡觉拿来学习拿来锻炼，都是有益无害。这么算下来真是赚了不少，条件虽然不是特别舒适，但是比起每天浑浑噩噩地地铁来地铁去，总要舒服许多。

对金钱这个东西，我一直看得不是太重，不会刻意挥霍也不会如何去想着节约。平时花钱纯看心情，吃好点买好点，只要自己心里舒畅，绝对不犹豫。毕竟如果只是吃喝用，现在我还是能把自己养得好好的。之前买macbook，买iPhone，不是想说明自己有多少钱，或者在别人面前装，而是体验过了，确实好用，那么就买了，而实际上买完他们我也差不多没什么存款了，但是我依然觉得很值。

近10年来国内的媒体和电视剧乐此不疲地宣扬金钱至上、拜金主义，整个社会风气都弥漫着一股铜臭味，他们“无意识地”宣扬纸醉金迷，描述各种“有钱人的生活”是多么其乐无穷。而他们的目标受众，经过多年培养，也就成长为了拆散无数情侣的“女方家长”。“有房有车”成为了各种结婚恋爱的前提，这一点可以理解，希望自己子女今后能不吃苦，过得上好的生活，但是这不代表需要“马上”过上这种生活，试问当年你们结婚的时候，有一台缝纫机是不是也觉得幸福满满呢。我从小到大在政治书上学到最有用的一句话就是“以发展的眼光看待问题”，如今大家的生活节奏都变快了，很多人都只关注当下，仿佛现在就等于将来。月薪2000的人一直埋怨，房子2W一平，买得起房子要工作100年；女儿喜欢上了一个穷小子，好多年都要受穷受苦；但是他们有没有想过，今年你2000月薪，明年还会是吗，你不会进步不会提升吗？如果你只是这么混，那么你工作25年也不是25年经验，而是一年的经验用了25年。同样的，当初马云和他妻子结婚时，他也是处处碰壁；李安导演潜心几年学电影，他老婆养了他几年……

小时候邻里亲戚关系都很和睦，因为大家都穷，穷得只剩下人情味。现在大家的财富都参差不齐地累积了上来，喜欢了互相攀比，喜欢“脸上有光”的感觉，人情味渐渐地被腐蚀掉了，或者说是变味了。所以有时我很矛盾，在外地时非常想念自己的家乡，但在家待久了，家里的人情世故还是有些让我招架不住。虽说是传统，传统需要尊重，但是时代在变化，在我们家乡小县城，传统明显做不到“与时俱进”，因为这些都已经根深蒂固了。

给自己设定了几个阶段，混沌期、积累期、收获期，混沌期在经过大学四年的洗礼已经度过，积累期多学一些多懂一些，摸索一下自己今后想走的方向，当时机成熟了，我想我需要做点有价值的事情，顺便赚点钱了。

最后分享一首北岛著名的诗，相信很多人都看过或听过，《波兰来客》

**那时我们有梦 **

**关于文学，关于爱情 **

**关于穿越世界的旅行 **

**如今我们深夜饮酒 **

**杯子碰到一起 **

都是梦破碎的声音

以此自省，梦要么继续做要么终结，但破碎不应该是其终结的方式

 

一则故事

发表于 2014-10-28 更新于 2016-09-21 分类于 纪录 本文字数： 161 阅读时长 ≈ 1 分钟

网上看到一个故事，死神让一对情侣两个人之中只能有一个活下来，谁生谁死由双方石头剪刀布决定。

男孩和女孩约好，两个人都出石头，结果男孩出了剪刀，女孩出了布。

当时这个故事的结论叫：算计了别人，却最终算计了自己。

但我的结论是，男孩因为爱这个女孩，希望牺牲自己；而女孩太了解男孩，知道他一定会出剪刀，而选择了布…

始终相信世界上美好之物多于丑恶，这也让我们自己变得更好。

最近很火的冰桶挑战

发表于 2014-08-21 更新于 2016-09-21 分类于 纪录 本文字数： 768 阅读时长 ≈ 3 分钟

这两天漫天充斥着“冰桶挑战”、“ALS”的词汇，突然冒出了一批国内明星大腕们拎起水桶往自己头上淋冰水的视频。细细一搜才知道，原来是美国肌萎缩性脊髓侧索硬化症协会（ALS Association）筹款而发起的活动。参加者需要让一个注满冰块及水的水桶倒在自己头上，并且将整个过程拍成视频、并上传至社交网络。另一方面，参加者亦会提名三名其他人士、要求其仿效，被提名者需于24小时回应，并可选择接受相同挑战，或是改成捐款100美元给慈善团体作为代替，或是两者都做。另外已接受挑战者的捐款金额起步点亦可由100美元降至10美元。至少刚说到开始，我觉得这个活动就已经非常成功了，因为通过这些“有意思的视频”，我第一次知道了世界上还有这种病。

至于为什么突然在中国一夜爆火，我想这就是社交媒体的力量把：有影响力的人通过有影响力的平台，产生病毒式的传播。而相比于之前《Baby》的Justin Bieber以及《江南Style》的鸟叔，只能说有过之而无不及。一来这是公益事业，正能量满满；二来被点名的人必须在24小时内回应，时效性非常强，也难怪百度指数会呈现下面的形状：

 

近两天百度的相关搜索量呈爆发式的增长，但是另外结果也很显然，关注冰桶的人远高于关注ALS的人，这是完全可以理解的现象。这本身是一个形式大于内容的活动，但是更值得肯定的是，关注ALS的人也增加了不少。网上有许多网友点评某某某作秀，借势操作……在有微博以来的历史中，这绝不是个例，而是一个普遍现象。有些人一开始怀着偏见做了类似的评论，而看到这些评论的人关注的点也被自然而然地带跑了，在这类评论中似乎找到了自己的“话语权”，也转化成了“批评家”。对此我想到了鲁迅先生一句话：“不惮以最坏的恶意来推测中国人”，而到了现在我们很多时候都会以恶意去揣测他人的想法，而忽略了他人的行为的正面意义。对于敢于接受挑战的人，我们还是应该报以掌声，不管他是不是为了炒作，不管他有没有捐钱，甚至不管他的水里到底有没有加冰……但是至少，他们的加入扩大了这个活动的影响力。光是这点，就不应该遭到太多的质疑与批评。生活本是如此艰辛，何必拒绝这一记冰爽的正能量呢？

一次不愉快的购书体验

发表于 2014-06-18 更新于 2016-09-21 分类于 纪录 本文字数： 748 阅读时长 ≈ 3 分钟

早在6月1日，京东就开始漫天做广告了，各种打折促销。而在其中6月8号当天，各种图书5折。于是我也屁颠屁颠地跑去贪便宜了，买点“心灵鸡汤”的书看看顺便想买一套最近比较喜欢的美剧《冰与火之歌》的原著来读一读。一卷150，半价也要75，心想都5折了，咬咬牙买了把，支持下原作者。

书一到迫不及待地拆开，但是迎来的就是两个字：失望。纸张质量差的我不想谈了，比路边摊卖的盗版还要差，从侧面摸起来就感觉毛毛糙糙，翻开以后除了一些刺鼻的怪味，纸张也又薄又糙，这对我来说是难以忍受的。于是很快我申请了换货，要说京东让人比较满意的地方，我觉得客服MM和配送人员绝对是亮点。客服反应很快，态度很好，听了我的抱怨没有任何解释而是立即帮我安排换货；配送人员之后动作也很快，带来了新书给我。虽然“新书”的质量依然毫无改善，但是也没有让我感到恼火，毕竟他们也不能把控商品的质量，而是劝我如果不满意就直接退货。

退货我觉得不是这个事情解决的终点，更多的是我要去商品页面做个评价让别人知道，但是京东竟然没有对这本书的店家做评价的地方，要么是整体的书评，要么是对本次购物的评价。本次购物的评价内容有三项：对包装的满意度，对配送速度的满意度，配送人员服务满意度。质量的选项看都看不到，真是奇葩的产品逻辑！还是说本身就心虚，不想让人来评价？于是乎我跑到整体书评的地方去发表一下感慨，洋洋洒洒书写了200多字的不满。在差评拦里，发现还有数百位朋友跟我一样的情况，真是悔不当初，没有先来看看差评具体是怎么说的。再多的好评感觉都没用，很早就听说JD会自己灌好评进去，就同我之前会通过SQL在我文章的评论处灌一些好玩的评论一样。但我是图个好玩你是做生意！淘宝要刷评价至少还有点难度，得都是真实用户，收取费用帮忙评价，jd因为是你自己家的店，想怎么灌怎么灌。

以后网购，先看差评，数量少的话可以忽略，数量如果够多(比如大于100)，用户层次丰富的话，就算99%的好评率，我也宁愿去相信那1%。信任建立起来很难，崩塌却很容易，且行且珍惜。

 

“全民英雄”《哥斯拉》

发表于 2014-06-17 更新于 2016-09-21 分类于 纪录 本文字数： 371 阅读时长 ≈ 1 分钟

两天前跑到了久违的电影院看了一场《哥斯拉》，我作为这只变种巨蜥的“粉丝”已有10多年了，还记得小时候家里租了一张盗版碟，让.雷诺版的《哥斯拉》，在那个年代看这种片子感觉非常神奇，能把这种巨兽和场景做得如此逼真。所以当得知新版哥斯拉3D要上映时，我觉得非常有必要再体验下这个怪兽破坏世界的震撼场景了……

看完之后有些失望，因为这个感觉不是我心目中怪兽的霸主，而是一个万年单身看不过另一对怪兽男女秀恩爱繁殖后代因而把它们干掉顺便不小心拯救了地球的故事。不是人类和哥斯拉的战斗，而是《迪迦奥特曼》类型的怪兽大战，两只长得特别高大的怪物在林立的大楼中间拳打脚踢，人类则在旁边看戏。印象中不可一世的怪兽之王变成了拯救世界的上古之神，结局看似很正面却没有惊喜，比起1998年版的哥斯拉，除了3D效果以外，感觉其他的都比不上后者。

稍微地吐槽了一下，不过3D效果还是需要赞一个，毕竟比起某些3D电影其实就是看3D字幕的来说，它已经显得诚意满满了。

Universal Analytics监测细节的完善(七)——定时发送数据

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics 本文字数： 685 阅读时长 ≈ 2 分钟

除了及时发现服务器端或者数据的异常之外，更多的情况下我们会需要定期知道一些特定的数据。作为网站分析人员，很可能会收到其他各个部门人员的需求，“给我导个XX数把”，“我们部门需要知道一下XXXX的数据”，这个时候千万别愣头愣脑的来一个需求满足一个，应该首先明确一下，需要这些数字的目的是什么？这个需求是一次性的还是长期的？网站分析更多的时间应该侧重在分析这块，而不是帮忙跑数据。所以如果是一次性的需求，那么友情赞助一下，没有问题；如果是长期的，写清楚需求，哪些维度，哪些指标，时间间隔，是否细分等等；如果他也答不上来要这些数字干吗，那还是算了把……

稍微有点跑题，我们回到刚才的问题上来，如果是一个长期的、定期的数据需求，只要明确了上面所要求的点，我们是可以一劳永逸地实现这个功能的，只需用到UA的定时发送数据功能。

在UA的Reporting界面中，我们可以看到几大类的数据报告，Audience(用户),Aquisition(流量获取), Behavior(行为),Conversion(转化)这几个主要大类涵盖了100多份报告，可以从一个完整的角度诠释你网站运营的整体状况，大部分报告界面上方都有一个Email按钮，点击该按钮就能将你当前视图中的数据以Email的形式定期地发送到你指定的邮箱中。

但是注意，这觉不仅限于默认提供的100多个报表，因为多个维度的交叉会使得报告的可能性成几何倍数增长。所以若要满足相应需求，需要用到自定义报告。

在界面顶部导航中选取Customization,点击New Custom Report，即可新建自定义报告

选定相应的维度和指标，点击保存即可。

但是有一点需要注意，维度和指标之间是有对应的关系的，而不是你想怎么选就怎么选。在了解各个维度和指标的意义及使用场景的前提下，注意可能的上下级和对应关系，因为如果他们之间出现了逻辑错误的话，自定义报告会是一片空白，这个时候请回去仔细查看你的设置，如果不是很清楚的话，可以多尝试几次，相信最后一定会有结果的。

Universal Analytics监测细节的完善(六)——数据警报

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics 本文字数： 426 阅读时长 ≈ 2 分钟

上一篇介绍了后端状况的监控设置，这一次我们需要通过设置对数据的异常情况得到比较及时的反馈。

一般广告公司或互联网公司运营部门，都是整天和数据打交道的。而其中一项很重要的职责是对于访问数据中的异常得到非常及时的反馈并找到针对性的措施解决相应的问题。而在广告公司更不幸运的是，有时候你一个人会管理几个甚至十几个网站的GA账户，而未必能每天都对着GA账户一个一个地去看，那么怎么才能比较高效地在这么多账户中及时地发现问题呢？这就需要用到GA的Alert预警功能。

打开UA后台，找到Admin-View-Custom Alert

如图按提示填写相应的内容，选定相应的维度和指标，当指标符合某类特定条件时(绝对值大于或小于特定值,变化的比率大于或小于特定值)，则向你发送邮件预警，提醒你数据有比较明显变化，是该看看后台了。(希望我能收到某天100 session的警告)

为了让用户更加方便地使用这个功能，Google非常人性化地加上了重复关联(图中的Apply)的功能，即如果你需要监控一系列站点的同一类数据，那么你只需设置一次，然后关联到其他的账户中，如果出现问题发送预警，GA会在邮件中以列表的形式呈现出来，非常方便。

Universal Analytics监测细节的完善(五)——网站服务器是否正常 - 监控宝

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics 本文字数： 732 阅读时长 ≈ 3 分钟

除了数据之外，你的网站服务器是否稳定也是一个非常重要的因素。作为个人站长来说，不可能每时每刻甚至说每天去查看自己的网站，那么我们就需要一些工具或者平台的辅助，定时地帮我们测试网站访问是否正常(而且我们还不想花钱),那么**监控宝**是你非常好的选择。

监控宝是由北京云智慧科技所提供的一项非免费服务，可以定时监控网站、服务和服务器器的可用率（Uptime）和响应时间(Response Time)，一旦网站无法连结、Apache等Web服务器负载过高、Mysql等数据库压力过大、或是服务器发生错误，监控宝服务就会在最短的时间内以Email或简讯来通知网站管理员，让因网站停摆而造成的损失降到最低。监控宝同时支持基于线路和地区的监控告警设置，可以用来较好的应对中国复杂的网络状况和环境，帮助网站管理员更全面的了解和管理网站。

除此之外，该服务也提供实用的「使用者访问速度追踪」功能，可以追踪国内不同地区的使用者进入你网站的速度。
对免费用户，监控宝提供5个监测需求，你只需输入你需要监测的请求或者链接，设置好相应的时间间隔，那么监控宝服务器则会按照设置的时间间隔无限地进行请求并返回结果，若某一次请求失败或者超时，则会立刻发送短信或者E-mail通知你，最大限度地减少服务器问题带来的影响和损失。

我使用的是免备案的香港服务器，相对于内地服务器来说部是那么地稳定。所以更加需要这类的服务，因为这家供应商及客服态度非常好，只要你收到监控宝的邮件说打不开，就可以去找他们聊天了。

另外如果你或者你的企业有需要的话，可以购买监控宝的收费服务，反应更快，功能更强，其中一大亮点是可以通过授权同步GA中的数据，可以将访问、负载情况与流量数据进行结合，这个是非常有趣的事情，感兴趣的朋友可以试一试。对于企业来说，最贵的套餐也不过￥599/月，确实不贵。

用了他家免费的功能好几年，一直没有给与经济上的支持，所以在这里相当于给他打一个广告把，虽然我的博客影响力及其有限，但是轻声细语也可以是感谢，对吧。

Universal Analytics监测细节的完善(四)——监测站内搜索

发表于 2014-06-08 更新于 2016-09-21 分类于 Universal Analytics 本文字数： 433 阅读时长 ≈ 2 分钟

站内搜索一直是一块非常有价值的资源，也是容易被大多数人遗忘的宝藏。你的网站内容结构好不好，是否满足用户的需求，如何发掘用户其他潜在的需求，在用户的搜索内容中会得到很多的体现。所以，监测好用户站内搜索的内容是网站分析中不可或缺的一环。

那么在UA中如何设置站内搜索的监测呢？其实也比较简单，其实在GA中就早已有此功能，因为不起眼所以很少获得关注。下面我们来设置一下：

首先，确定搜索参数

如同添加默认识别搜索引擎一样，需要监测站内搜索的内容，首先你需要知道你搜索的内容存在什么位置、什么参数中。举我的网站作为例子，当我搜索google analytics的时候，搜索结果页的URL为：

http://jackie.ms/?s=google+analytics&submit=%E6%90%9C%E7%B4%A2

其中s就是我需要的参数 (补充一点，Wordpress站点的搜索参数都未s)

其次，后台补充设置

打开Admin-View-View Setting

打开Site Search Tracking，并将该参数填入相应位置，保存即可。
(最后选项为在content报告中URL是否去除搜索词参数)

补充，数据查看

在Reporting报告界面中，找到Behavior-Site Search即可查看相应站内搜索词，从这个时刻开始，你会更加了解你的用户，他们喜欢看什么内容，还想看什么内容，然后试着去补充这方面的内容把，相信用户会更加喜欢你的网站的。
